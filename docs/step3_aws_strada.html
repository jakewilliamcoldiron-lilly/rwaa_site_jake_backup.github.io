Step 3 - AWS / STRADA Access

⏱ Time: 45 minutes | ⏳ Approval: 10–14 business days

AWS STRADA is the cloud-based data warehouse hosting all RWE databases. All analysts require access.

Pre-requisite Training

Complete all four before requesting access:

  • Course 9049927: DEV-HO: RWE 301-ODM — Conducting Observational Studies
  • Course 9219012: DEV-HO: RWE 301a-ODM — PROCEDURE FOR Observational Studies (supplement)
  • Course 9277801: EC-PROC: Global Developing Real World Evidence (required specifically for AWS approval)
  • Course 9183089: DEV-HO: RWE 201-ODM — Applying RWD for RWE Generation

Create Encrypted Password File and Test Connection:

Open the file at the path above (or download here), enter your Redshift password where indicated, run all steps, then close without saving.

This creates two files in /home/&sysuserid/pswd/:

  • encrypted_redshift_pw.txt — your encrypted Redshift password, stored in SAS-encrypted format
  • MyEncryptedPwd.sas — a SAS macro file that reads the encrypted password and assigns it to a macro variable (by default, &rspw)
Macro Variable Naming

The macro variable name rspw used in MyEncryptedPwd.sas is a user-defined convention, not a system requirement. You may rename it to any valid SAS macro variable name (e.g., &redshift_pw, &my_pwd). However, the name used in the password generation program must match exactly in all subsequent programs that reference it. For example, if your encrypted password file assigns the password to &rspw, then every %include and libname or connect to statement must also use &rspw. Inconsistent naming will cause authentication failures at runtime.

Test the connection:

# SAS Code - run in SAS Enterprise Guide, not R
# %include "/home/&sysuserid/pswd/MyEncryptedPwd.sas";
# proc sql;
#   connect to ODBC as Redshift
#     (dsn=Access_Redshift_Prod password="&rspw" user=&sysuserid);
#   create table Redshift_schemas_I_can_see as
#     SELECT * from connection to Redshift
#     (select distinct schemaname from pg_catalog.pg_tables
#      where schemaname like 'rwe_strada_%'
#      union
#      select distinct schemaname from svv_external_schemas
#      where schemaname like 'rwe_strada_%');
# quit;

SAS Enterprise Guide — Citrix Apps and Profiles

Two SAS Enterprise Guide applications are available in Citrix, each serving a different primary purpose:

Citrix App Primary Use Notes
SAS_EG-PROD RWAA / STRADA analysis Default app for all RWAA real-world data work.
CLUWE SAS Enterprise Guide 8 Clinical development More stable release; can also be configured for RWAA server connections.

SAS_EG-PROD — Server Profiles (RWAA Default)

The following profiles are pre-configured or should be added in SAS_EG-PROD:

Profile Name Server Port Use
SAS_AWS_PROD gridrp9meta1.aws.lilly.com 8561 AWS STRADA production — primary RWAA work
SAS RHEL8 gridmetaprd1.am.lilly.com 8564 CLUWE / RHEL8 — RWAA and CLUWE data
SAS RHEL9 gridrp9mgr01.aws.lilly.com 8561 AWS RHEL9 environment

CLUWE SAS Enterprise Guide 8 — Server Profiles (Clinical + Fallback)

The following profiles are pre-configured or should be added in CLUWE SAS Enterprise Guide 8:

Profile Name Server Port Use
SAS Grid in AWS Cloud PROD gridrp9meta1.aws.lilly.com 8561 AWS STRADA — same server as SAS_EG-PROD
SAS RHEL9 gridrp9mgr01.aws.lilly.com 8561 AWS RHEL9 environment
Fallback — Use CLUWE SAS EG 8 If SAS_EG-PROD Is Unavailable

If SAS_EG-PROD encounters connectivity issues or fails to launch, you can use CLUWE SAS Enterprise Guide 8 as an alternative. Because SAS Enterprise Guide supports multiple server profiles, the RWAA servers (e.g., gridrp9meta1.aws.lilly.com:8561) can be added to this app as well. This provides a reliable fallback without waiting for SAS_EG-PROD to be restored.

DSN Reminder

Use dsn=Access_Redshift_Prod when connected to an AWS server (port 8561). Use dsn=Access_Redshift when connected to a CLUWE/RHEL8 server (port 8564). These are not interchangeable.

Setting the STRADA path in SAS:

# SAS Code - run in SAS Enterprise Guide
# %let PROGPATH = /veo_fsx/strada/projects/rwe_all/<your_project_folder>;
# libname val "&PROGPATH./Validation";
GPS Users

You have read-only access to rwe_strada_wrk. Use rwe_strada_gps_wrk schema instead. On CLUWE profile (not AWS PROD), use dsn=Access_Redshift instead of Access_Redshift_Prod.
